## 第 1 页：课题背景与研究目标

### 标题
**基于注意力网络的微表情识别：阶段性进展汇报**

### 页面内容
1. 微表情的定义：持续时间短、幅度小、难以人工识别。
2. 课题目标：识别面部细微肌肉运动对应的情绪类别。
3. 当前方法路线：
   - 数据集：CASME II
   - 特征：onset/apex 光流
   - 模型：双分支注意力网络
   - 评估：LOSO

### 你可以直接说
> 我的毕设方向是基于注意力网络的微表情识别。由于微表情持续时间短、运动幅度很小，直接做静态图像分类效果往往不稳定，因此我当前采用的路线是：先从关键帧中提取光流特征，再用带注意力机制的双分支网络进行分类，并用 LOSO 评估跨被试泛化能力。

### 导师可能追问
- 为什么不用普通 RGB 图像直接分类？
- 为什么任务难？

### 回答思路
- 微表情信号很弱，RGB 容易被身份、光照、背景干扰；
- 光流能更直接描述微小运动。

## 第 2 页：数据集与标签设计

### 标题
**数据集内容、分类方式与适用性分析**

### 页面内容
1. 当前使用数据集：CASME II。
2. 数据内容：原始图像序列 + Excel 标注（Subject / Filename / OnsetFrame / ApexFrame / Emotion）。
3. 当前分类方式：4 分类。
   - positive
   - negative
   - surprise
   - others
4. 为什么做 4 分类而不是更细分类：
   - 样本量小；
   - 细分类更难训练；
   - 先搭可运行 baseline。
5. 数据集局限：
   - 样本少；
   - 类别不均衡；
   - 泛化差。
6. 可选扩展数据集：SAMM / SMIC / CAS(ME)^2。

### 你可以直接说
> 当前我先使用 CASME II 数据集，因为它提供了 onset 和 apex 标注，便于实现基于光流的微表情识别流程。现阶段采用 4 分类而不是更细粒度分类，主要是考虑到微表情数据集样本量较小、类别分布不均衡，先降低任务难度，把完整流程搭起来。这个数据集适合做 baseline，但不足以支撑很强的泛化结论，后续如果条件允许，我希望补充更多数据集或做跨库实验。

### 导师可能追问
- 为什么不用更大的数据集？
- 4 分类会不会太粗？

### 回答思路
- 当前优先解决“可运行、可解释、可复现”；
- 4 分类是工程上对小样本问题的折中。

## 第 3 页：当前技术路线与数据预处理

### 标题
**预处理流程：从关键帧到光流特征**

### 页面内容
建议放一张流程图：

`Excel 标注 → 定位样本 → 读取 onset/apex → resize 128×128 → 计算 TV-L1 光流 → 保存 u/v .npy`

然后配 3 个关键词解释：

1. `onset`：微表情开始帧；
2. `apex`：微表情峰值帧；
3. 光流：描述像素运动方向和幅度。

再加一句：

- `u`：水平光流
- `v`：垂直光流

### 你可以直接说
> 当前预处理阶段，我先读取 CASME II 的 Excel 标注，获得每个样本对应的 onset 和 apex 帧，然后读取这两帧图像并统一缩放到 128×128。之后使用 TV-L1 稠密光流提取两帧之间的运动信息，并将水平与垂直两个方向的光流分别保存为 `.npy` 文件，作为模型输入。这样做的核心考虑是：微表情的判别信息主要体现在细微运动上，光流比直接使用 RGB 更有针对性。

### 导师可能追问
- 为什么只用两帧？
- `.npy` 是什么？
- 水平/垂直光流是什么？

### 回答思路
- 两帧方案实现简单，适合作为 baseline；
- `.npy` 是 NumPy 数组，便于高效保存矩阵；
- `u/v` 表示像素在水平方向和垂直方向的位移。

### 这一页最后要主动承认的不足
- 目前还没做人脸对齐；
- 只用两帧，时序信息不完整；
- 后续可改成多帧建模或局部区域建模。

## 第 4 页：模型结构与训练方式

### 标题
**模型设计：双分支注意力网络 + LOSO 训练**

### 页面内容
可以画一个很简化的结构图：

- 输入 1：`u` 光流
- 输入 2：`v` 光流
- 两个分支分别做：Conv + BN + ReLU + CBAM + Pool
- 特征拼接
- 全连接分类
- 输出 4 类

旁边补一列超参数：

- batch size = 32
- epochs = 60
- optimizer = AdamW
- lr = 0.001
- scheduler = CosineAnnealingLR
- validation strategy = LOSO

### 你可以直接说
> 当前模型采用双分支注意力网络，一条分支处理水平光流，另一条分支处理垂直光流。每条分支内部使用卷积提取局部特征，并加入 CBAM 注意力模块，让网络更关注重要通道和关键空间区域。两条分支的特征在后续进行拼接，再通过全连接层完成 4 分类输出。训练方式采用 LOSO，也就是留一受试者交叉验证，用于评估模型对不同个体的泛化能力。

### 导师可能追问
- CBAM 是什么？
- 为什么不用 YOLO/ResNet？
- 为什么这样设置卷积和池化？

### 回答思路
- CBAM 是通道注意力 + 空间注意力；
- 当前任务是分类，不是目标检测；
- 该结构是适合小数据集的轻量 CNN baseline。
